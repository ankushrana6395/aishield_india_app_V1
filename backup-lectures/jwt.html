Understanding JWT Structure

JWT Structure Ki Samajh

JWT (JSON Web Token) is an open standard for securely transmitting information between parties as a JSON object. It's commonly used for authentication and information exchange.

JWT (JSON Web Token) partiyon ke beech sanskriti ko surakshit tarike se prasaran karne ke liye ek khol maanak hai. Iska upyog aksar pramaanikaran aur sanskriti aadano-pradano ke liye kiya jata hai.

Real Analogy: Like a sealed envelope with your ID card inside that can be verified without opening it.

Asli Example: Ek band lipta liye ke andar aapki ID card ke saath ek sealed envelope ke saman jo ise kholne ke bina hi satyaapit kiya ja sakta hai.

A JWT consists of three parts: Header, Payload, and Signature, separated by dots.

Ek JWT mein teen bhag hote hain: Header, Payload, aur Signature, jo dots ke dwara alag kiye jate hain.

JWT is critical because it's used for authentication in most modern web applications.

JWT bahut khatarnaak hai kyunki yeh adhikansh aadhunik jaal anuprayogon mein pramaanikaran ke liye upyog kiya jata hai.

JWT Components Breakdown

JWT Components Ki Vistrit Jankari

Header.Payload.Signature
  • Header: Contains metadata about the token, including the signing algorithm (e.g., HS256)
  • Header: Token ke baare mein metadata hota hai, jisme signing algorithm (jaise, HS256) shaamil hai
  • Payload: Contains the claims (user data, permissions, etc.)
  • Payload: Davaon ko shaamil karta hai (upyogakarta sanskriti, anumatiyan, aadi)
  • Signature: Ensures the token hasn't been tampered with
  • Signature: Yeh sicher karta hai ki token ke saath chhedchhad na kiya gaya ho

Advanced Bypass Techniques

Uchatam Bypass Tarika

1. None Algorithm Bypass

When the server accepts "none" algorithm, you can create a token without a signature.

Jab server "none" algorithm ko svikar karta hai, toh aap bina signature ke ek token bana sakte hain.

Example: 1. Header: {"alg": "none", "typ": "JWT"} 2. Payload: {"user": "admin", "iat": 1516239022} 3. Signature: (empty - just a dot at the end) Final Token: eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.eyJ1c2VyIjoiYWRtaW4iLCJpYXQiOjE1MTYyMzkwMjJ9.

2. Algorithm Confusion (RS256 to HS256)

Exploit when the server uses the public key as the secret for HMAC signing.

Jab server HMAC signing ke liye sarvajanik kunji ko rahasya ke roop mein upyog karta hai toh iska faida uthayein.

Steps: 1. Obtain the public key from the server (usually in /jwks.json) 2. Change algorithm from RS256 to HS256 in header 3. Modify payload to change user to "admin" 4. Sign with the public key as the secret

3. Key Confusion Attack

When the server uses the same key for multiple purposes, you can trick it into using the wrong key.

Jab server ek hi kunji ko anek uddeshyon ke liye upyog karta hai, toh aap ise galat kunji upyog karne ke liye dhokha de sakte hain.

Example: 1. Server uses the same key for both signing and encryption 2. Trick the server into using the encryption key for signing 3. Forge tokens with the encryption key

4. Brute Force Secret

When HS256 is used with a weak secret, you can brute force it using tools like John the Ripper.

Jab HS256 ek kamzor rahasya ke saath upyog kiya jata hai, toh aap John the Ripper jaise upakaranon ka upyog karke ise brute force kar sakte hain.

Steps: 1. Extract the JWT token 2. Use john or hashcat to brute force the secret 3. Once found, forge new tokens with any payload

Real-World Case Studies

Vastavik Duniya Ke Udaharan

Case Study 1: Social Media Platform

Scenario: A social media platform used JWT for user authentication with a weak secret.

Paristhiti: Ek samajik media manch ne upyogakarta pramaanikaran ke liye kamzor rahasya ke saath JWT ka upyog kiya.

Exploitation: Used hashcat to brute force the secret and forge admin tokens.

Doharpan: Rahasya ko brute force karne ke liye hashcat ka upyog kiya gaya aur prashashan tokens forge kiye gaye.

Impact: Full account takeover of any user, including admins.

Prabhav: Kisi bhi upyogakarta, prashashanon ke saath, ka poori khaate par nyotpan.

Case Study 2: E-commerce API

Scenario: An e-commerce API accepted "none" algorithm in JWT tokens.

Paristhiti: Ek e-commerce API ne JWT tokens mein "none" algorithm ko svikar kiya.

Exploitation: Created tokens with admin privileges by removing the signature.

Doharpan: Signature ko hatakar prashashan sudharatmak adhikaron ke saath tokens banaye gaye.

Impact: Unauthorized access to admin functions and customer data.

Prabhav: Prashashan kaaryon aur graahak sanskriti tak anadhikrit pravesh.

Secure Coding Practices

Surakshit Coding Abhyas

1. Use Strong Secret Keys

Always use cryptographically strong, randomly generated secrets for HMAC signing.

Hamesha HMAC signing ke liye cryptographic strong, yadrichchhik roop se utpann rahasyaon ka upyog karein.

// Example: Generate a strong secret const crypto = require('crypto'); const secret = crypto.randomBytes(32).toString('hex');

2. Validate Algorithm

Never allow the "none" algorithm and explicitly specify allowed algorithms.

Kabhi bhi "none" algorithm ki anumati mat dein aur spasht roop se anumati prapt algorithmon ko nirdisht karein.

// Example: Validate algorithm const allowedAlgorithms = ['HS256', 'RS256']; const tokenHeader = JSON.parse(atob(token.split('.')[0])); if (!allowedAlgorithms.includes(tokenHeader.alg)) { throw new Error('Invalid algorithm'); }

3. Proper Key Management

Use separate keys for signing and encryption, and rotate them regularly.

Signing aur encryption ke liye alag kunjiyon ka upyog karein, aur unhe niyamit roop se ghumate rahen.

Best Practices: 1. Use different keys for signing and encryption 2. Rotate keys periodically 3. Store keys securely (e.g., HSM, environment variables) 4. Use key IDs (kid) to identify keys

4. Implement Token Expiration

Always set short expiration times and implement refresh tokens for long sessions.

Hamesha chhoti samayavadhi set karein aur lambi dakhle ke liye refresh tokens ko laagoo karein.

// Example: Set expiration time const payload = { user: 'admin', iat: Math.floor(Date.now() / 1000), exp: Math.floor(Date.now() / 1000) + (60 * 60) // 1 hour };
