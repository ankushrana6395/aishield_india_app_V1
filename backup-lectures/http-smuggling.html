The Proxy Mismatch Analogy

Proxy Mismatch Wala Scene

HTTP Smuggling occurs when two HTTP servers (typically a front-end proxy and a back-end server) interpret the same ambiguous HTTP request differently, allowing an attacker to "smuggle" requests through the front-end proxy to the back-end server.

HTTP Smuggling tab hota hai jab do HTTP servers (aam taur par ek front-end proxy aur ek back-end server) ek hi ambigous HTTP request ko alag tarike se interpret karte hain, jisse hamlawar front-end proxy se back-end server tak request "smuggle" kar sakta hai.

Real Analogy: Two security guards checking the same bag but interpreting what's inside differently - one sees a harmless item while the other sees a weapon.

Asli Example: Do security guards ek hi bag ko check kar rahe hain lekin usme jo hai uska alag alag interpretation kar rahe hain - ek ko lagta hai woh ek harmless cheez hai jabki dusre ko lagta hai woh ek weapon hai.

If exploited, HTTP Smuggling can lead to cache poisoning, session hijacking, bypassing security controls, and even complete system compromise.

Agar exploit kiya jaye, toh HTTP Smuggling cache poisoning, session hijacking, security controls ko bypass karna, aur poori system compromise tak le ja sakta hai.

HTTP Smuggling is critical because it exploits discrepancies in HTTP parsing between servers.

HTTP Smuggling bahut khatarnaak hai kyunki yeh servers ke beech HTTP parsing mein discrepancies ka faida uthata hai.

Advanced Bypass Techniques

Uchatam Bypass Tarika

1. Header Obfuscation

When standard headers are blocked, obfuscate them with case variations or spaces.

Jab standard headers rukawat ho jate hain, toh unhe case variations ya spaces ke saath obfuscate karein.

Obfuscation Techniques: - Transfer-Encoding: chunked - transfer-encoding: chunked - Transfer-Encoding : chunked - Transfer-Encoding: xchunked - X-Transfer-Encoding: chunked

2. Encoding Bypass

Use different encodings to bypass normalization filters.

Normalization filters ko bypass karne ke liye alag encodings ka upyog karein.

Encoding Techniques: - Tab characters: Transfer-Encoding:\tchunked - Newline characters: Transfer-Encoding\n: chunked - Unicode characters: Transfer-Encoding: chūnked - Hex encoding: Transfer-Encoding: %63%68%75%6e%6b%65%64

3. Chunk Size Manipulation

Manipulate chunk sizes to confuse parsers.

Parsers ko confuse karne ke liye chunk sizes ko manipulate karein.

Chunk Manipulation: - Zero padding: 0000\r\n - Hex variations: 0x0\r\n - Extra whitespace: 0 \r\n - Multiple zeros: 00\r\n

4. Multi-part Request Smuggling

Combine smuggling with multi-part requests to bypass more sophisticated filters.

Adhikatam filters ko bypass karne ke liye smuggling ko multi-part requests ke saath combine karein.

Multi-part Smuggling: POST /upload HTTP/1.1\r\n Host: example.com\r\n Content-Type: multipart/form-data; boundary=----WebKitFormBoundary\r\n Content-Length: X\r\n Transfer-Encoding: chunked\r\n \r\n 0\r\n \r\n ------WebKitFormBoundary\r\n Content-Disposition: form-data; name="file"; filename="test.txt"\r\n Content-Type: text/plain\r\n \r\n Smuggled content here

Secure Coding Practices

Surakshit Coding Abhyas

1. Strict Header Parsing

Implement strict parsing of HTTP headers and reject ambiguous requests.

HTTP headers ka kathor parsing lagayen aur ambigous requests ko reject karein.

// Example: Reject requests with both Content-Length and Transfer-Encoding if (request.hasHeader("Content-Length") && request.hasHeader("Transfer-Encoding")) { response.setStatus(400); return; }

2. Consistent HTTP Parsing

Ensure front-end and back-end servers use the same HTTP parsing library.

Front-end aur back-end servers ek hi HTTP parsing library ka upyog karein.

// Example: Use the same HTTP library across all components // Front-end proxy and back-end server both use Apache HttpComponents // This ensures consistent interpretation of requests

3. Request Normalization

Normalize HTTP requests before processing to eliminate ambiguities.

Ambiguities ko khatam karne ke liye HTTP requests ko sansadhit karne se pahle normalize karein.

// Example: Normalize headers before processing String normalizedHeader = header.trim().toLowerCase(); if (normalizedHeader.equals("transfer-encoding")) { // Process according to standard }

4. Disable Chunked Encoding if Not Needed

If your application doesn't require chunked encoding, disable it completely.

Agar aapki application ko chunked encoding ki aavashyakta nahi hai, toh ise poori tarah nishkriya karein.

// Example: Disable chunked encoding in server configuration // Apache: HttpProtocol disableChunking="true" // Nginx: chunked_transfer_encoding off;
