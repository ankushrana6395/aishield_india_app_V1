The Real-Time Connection Analogy

Real-Time Connection Ka Scene

WebSocket vulnerabilities occur when attackers manipulate real-time connections — like eavesdropping on a private telephone line.

WebSocket kamzoriyan tab hoti hain jab attackers real-time connections ko manipulate karte hain — jaise private telephone line par eavesdropping.

If exploited, WebSocket vulnerabilities can lead to unauthorized message interception, data manipulation, and denial of service attacks.

Agar exploit kiya jaye, toh WebSocket kamzoriyan unauthorized message interception, data manipulation, aur denial of service hamlawar tak le ja sakti hain.

WebSocket vulnerabilities can expose sensitive real-time data and give attackers unauthorized access to communication channels.

WebSocket kamzoriyan sensitive real-time data ko expose kar sakti hain aur attackers ko communication channels tak unauthorized access de sakti hain.

Real-World WebSocket Vulnerabilities

Vastavik Duniya Ke WebSocket Vulnerabilities

Case Study 1: Cross-Site WebSocket Hijacking

Scenario: A chat application had vulnerable WebSocket origin validation.

Paristhiti: Ek chat application mein vulnerable WebSocket origin validation thi.

Exploitation: Used Cross-Site Request Forgery to hijack WebSocket connections.

Doharpan: WebSocket connections hijack karne ke liye Cross-Site Request Forgery ka upyog kiya gaya.

Impact: Exposed private chat conversations leading to information disclosure.

Prabhav: Information disclosure ke karan private chat conversations exposed ho gaye.

Case Study 2: DoS via WebSocket Flooding

Scenario: A gaming server had insufficient WebSocket connection limits.

Paristhiti: Ek gaming server mein insufficient WebSocket connection limits thi.

Exploitation: Flooded server with thousands of WebSocket connections simultaneously.

Doharpan: Server ko simultaneously thousands WebSocket connections ke saath flood kiya gaya.

Impact: Server became unavailable, disrupting service for legitimate users.

Prabhav: Server unavailable ho gaya, jo legitimate users ke liye service disrupt karta tha.

Secure WebSocket Practices

Surakshit WebSocket Abhyas

1. Strict Origin Validation

Validate origin headers and reject unauthorized connections.

Origin headers validate karein aur unauthorized connections reject karein.

// Validate origin on server side if (!isAllowedOrigin(request.headers.origin)) { socket.destroy(); return; }

2. Proper Authentication

Use secure session management and validate user privileges.

Surakshit session management ka upyog karein aur user privileges validate karein.

// Verify JWT token on connection const token = extractToken(socket.handshake.headers); if (!verifyToken(token)) { socket.disconnect('Unauthorized'); }

3. Rate Limiting & DDoS Protection

Implement connection limits and frame rate restrictions.

Connection limits aur frame rate restrictions implement karein.

// Rate limiting implementation if (connectionsPerIP.get(ipAddress) > MAX_CONNECTIONS) { socket.destroy(); return; }

4. Use WSS over WS

Always use secure WebSocket connections (WSS) in production.

Production mein hamesha secure WebSocket connections (WSS) ka upyog karein.

// Use WSS in production const io = require('socket.io')(server, { cors: { origin: ["https://yourdomain.com"], credentials: true } });
