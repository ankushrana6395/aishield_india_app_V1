Understanding Clickjacking

Clickjacking Ki Samajh

Clickjacking (UI Redress Attack) is a malicious technique where an attacker tricks a user into clicking on something different from what the user perceives, potentially revealing confidential information or taking control of their computer.

Clickjacking (UI Redress Attack) ek dushit technique hai jahan ek hamlawar upyogkartri ko uske samajh se alag kisi cheez par click karne ke liye bewakuf banata hai, jisse sanvedansheel jankari prakat ho sakti hai ya unke computer ka niyantran liya ja sakta hai.

Real Analogy: Like a fake button placed over a real button on a website - when you think you're clicking "Play Video", you're actually clicking "Transfer Money" hidden underneath.

Asli Example: Jaise ek website par asli button ke upar rakha gaya nakli button - jab aap sochte hain ki "Play Video" click kar rahe hain, aap actually mein niche chhipe "Transfer Money" par click kar rahe hain.

Clickjacking is dangerous because it can trick users into performing actions without their knowledge or consent, leading to account compromise, data theft, or financial loss.

Clickjacking khatarnaak hai kyunki yeh upyogkartriyon ko unki jaankari ya anumati ke bina kriyaen karne ke liye bewakuf bana sakta hai, jisse account compromise, data theft, ya financial loss ho sakta hai.

Clickjacking is also known as UI redress attack because it overlays hidden interfaces over legitimate ones.

Clickjacking ko UI redress attack bhi kaha jata hai kyunki yeh vaid interfaces ke upar chhipe interfaces overlay karta hai.

Types of Clickjacking Attacks

Clickjacking Attacks Ke Prakar

  • Classic Clickjacking: Tricking users into clicking hidden buttons or links
  • Classic Clickjacking: Upyogkartriyon ko chhipe buttons ya links par click karne ke liye bewakuf banana

  • Likejacking: Upyogkartriyon ko social media par pages like karne ke liye bewakuf banana
  • Cursorjacking: Manipulating cursor position to mislead clicks
  • Cursorjacking: Clicks ko gumrah karne ke liye cursor position manipulate karna
  • Cookiejacking: Stealing session cookies through deceptive actions

Real-World Case Studies

Vastavik Duniya Ke Udaharan

Case Study 1: Social Media Platform

Scenario: A major social media platform was vulnerable to likejacking attacks.

Paristhiti: Ek pramukh social media platform likejacking attacks ke liye kamjor tha.

Exploitation: Attackers created fake "video player" buttons that overlay the actual "Like" button, tricking users into liking pages unintentionally.

Doharpan: Hamlawaron ne nakli "video player" buttons banaye jo asli "Like" button overlay karte the, jisse upyogkartriyon ko anjaane mein pages like karne ke liye bewakuf banaya gaya.

Impact: Millions of users tricked into liking malicious pages, leading to widespread spam and phishing campaigns.

Prabhav: Lakshon upyogkartriyon ko dushit pages like karne ke liye bewakuf banaya gaya, jisse vyapak spam aur phishing campaigns hui.

Case Study 2: Banking Application

Scenario: A banking application had inadequate X-Frame-Options protection.

Paristhiti: Ek banking application mein aparyapt X-Frame-Options suraksha thi.

Exploitation: Attackers created fake banking pages that overlay the actual transfer confirmation buttons, tricking users into authorizing transactions.

Doharpan: Hamlawaron ne nakli banking pages banaye jo asli transfer confirmation buttons overlay karte the, jisse upyogkartriyon ko transactions authorize karne ke liye bewakuf banaya gaya.

Impact: Financial losses for customers and reputational damage to the bank.

Prabhav: Grahakon ke liye financial losses aur bank ko pratishtha nuksan.

Secure Coding Practices

Surakshit Coding Abhyas

1. X-Frame-Options Header

Use the X-Frame-Options HTTP header to control which sites can frame your content.

Yeh control karne ke liye ki kaunsi sites aapka content frame kar sakti hain, X-Frame-Options HTTP header ka upyog karein.

// Prevent all framing X-Frame-Options: DENY // Allow same-origin framing only X-Frame-Options: SAMEORIGIN // Allow from specific origin (deprecated but still used) X-Frame-Options: ALLOW-FROM https://example.com // Example in Apache config: Header always set X-Frame-Options "SAMEORIGIN" // Example in Nginx config: add_header X-Frame-Options SAMEORIGIN;

2. Content Security Policy (CSP)

Use CSP frame-ancestors directive for more granular control over framing.

Framing par adhik granular control ke liye CSP frame-ancestors directive ka upyog karein.

// Prevent all framing Content-Security-Policy: frame-ancestors 'none'; // Allow same-origin framing only Content-Security-Policy: frame-ancestors 'self'; // Allow from specific origins Content-Security-Policy: frame-ancestors example.com trusted-site.com; // Allow from specific origin with scheme Content-Security-Policy: frame-ancestors https://example.com; // Example in HTML meta tag (not recommended for CSP):

3. JavaScript Frame Busting

Implement JavaScript frame busting as a secondary defense mechanism.

Ek secondary bachav yantra ke roop mein JavaScript frame busting lagu karein.

// Basic frame busting if (top != self) top.location = self.location; // More robust frame busting // Check for framability if (window.self !== window.top) { // We're in a frame, take action window.top.location = window.self.location; }

4. Additional Protective Measures

Implement additional measures to protect sensitive actions from being framed.

Sanvedansheel kriyaon ko frame hone se bachane ke liye adhik suraksha upay lagu karein.

// 1. Use CAPTCHAs for sensitive actions // 2. Implement confirmation dialogs for important actions // 3. Use session-based tokens for state-changing operations // 4. Educate users about clickjacking risks // Example: Confirm sensitive actions function confirmAction(action) { return confirm(`Are you sure you want to ${action}?`); } // Use in forms:
