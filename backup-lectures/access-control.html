Understanding Access Control

Access Control Ki Samajh

Access control is a security mechanism that determines who can access what resources in an application. When broken, it allows unauthorized users to access data or functionality they shouldn't have access to.

Access control ek suraksha yantra hai jo yeh nirdharit karta hai ki kaun kis srotak tak pravesh kar sakta hai. Jab yeh toot jata hai, toh anadhikrit upyogkartri data ya functionality tak pravesh kar paate hain jiske unhe pravesh nahi hona chahiye.

Real Analogy: Like a building with multiple floors where each floor has different access permissions. If the access control system is broken, anyone can go to any floor regardless of their authorization.

Asli Example: Ek aise building ke jaise jisme kai manjilen hain aur har manjil ke alag-alag pravesh anumatiyan hain. Agar pravesh niyantrak pranali toot gayi hai, toh koi bhi vyakti kisi bhi manjil par ja sakta hai chahe uske paas anumati kyun na ho.

Broken access control is one of the most common and critical vulnerabilities in web applications, often leading to data breaches and privilege escalation.

Broken access control web applications mein sabse aam aur mahatvapurna kamjoriyan mein se ek hai, jo aksar data breaches aur privilege escalation ke liye neta hai.

Access Control is critical because it protects sensitive data and functionality from unauthorized access.

Access Control mahatvapurna hai kyunki yeh sanvedansheel data aur functionality ko anadhikrit pravesh se bachata hai.

The application checks the user's permissions before granting access to sensitive resources. When this check is missing or flawed, access control is broken.

Application sanvedansheel srotakon tak pravesh dene se pahle upyogkartri ke anumatiyon ki jaanch karta hai. Jab yeh jaanch gair-mojood ya kharaabi yukta hoti hai, toh pravesh niyantrak toot jata hai.

Types of Access Control

Access Control Ke Prakar

  • Vertical Access Control: Controls access based on user roles (admin vs user)
  • Vertical Access Control: Upyogkartri ke bhoomikaon ke aadhar par pravesh niyantrit karta hai (admin vs upyogkartri)
  • Horizontal Access Control: Controls access to other users' resources at the same level
  • Horizontal Access Control: Same level par doosre upyogkartiyon ke srotakon tak pravesh niyantrit karta hai
  • Context-Dependent Access Control: Controls access based on application state or business logic
  • Context-Dependent Access Control: Application avastha ya vyavasayik tark ke aadhar par pravesh niyantrit karta hai

Advanced Bypass Techniques

Uchatam Bypass Tarika

1. HTTP Method Override

When direct access is blocked, try changing HTTP methods or using method override headers.

Jab seedha pravesh rukawat ho jata hai, toh HTTP tarikon ko badalne ya tarika override headers ka upyog karein.

Common Method Override Headers: - X-HTTP-Method-Override: PUT - X-HTTP-Method: DELETE - X-Method-Override: PATCH

2. HTTP Header Manipulation

Use special headers to bypass access controls by impersonating privileged users or changing request context.

Vishesh headers ka upyog karke pravesh niyantrako bypass karein by garvit upyogkartiyon ka roop dharan karke ya anurodh sandarbh badalke.

Useful Headers for Bypass: - X-Original-URL: /admin - X-Rewrite-URL: /admin - Referer: https://admin.site.com/ - X-Forwarded-For: 127.0.0.1 - X-Remote-IP: 127.0.0.1

3. Parameter Pollution

Send multiple parameters with the same name to confuse access control mechanisms.

Pravesh niyantrak yantrikon ko bhramaane ke liye ek hi naam ke kai parameters bhejein.

Example: GET /api/user?id=123&id=456 Some backends only check the first parameter Others only check the last parameter

4. Path Traversal in URLs

Use path traversal techniques to access restricted endpoints.

Pratibandhit ant pointon tak pravesh karne ke liye path traversal tarikon ka upyog karein.

Example: Original: /api/user/123 Bypass: /api/../../../admin/user/123 Bypass: /api/user/123/.json

Real-World Case Studies

Vastavik Duniya Ke Udaharan

Case Study 1: Social Media Platform

Scenario: A social media platform allowed users to view any profile by changing the user ID in the URL.

Paristhiti: Ek social media platform upyogkartiyon ko URL mein upyogkartri ID badalke kisi bhi profile ko dekhne ki anumati de raha tha.

Exploitation: Used IDOR to access private profiles and messages of other users.

Doharpan: Doosre upyogkartiyon ke niji profiles aur sandeshon tak pahuchne ke liye IDOR ka upyog kiya gaya.

Impact: Massive privacy breach affecting millions of users.

Prabhav: Karodon upyogkartiyon ko prabhavit karne wala vishal gopaniyata ullanghan.

Case Study 2: E-commerce Platform

Scenario: An e-commerce site had an API endpoint for viewing order details.

Paristhiti: Ek e-commerce site mein aadesh vivaran dekhne ke liye API ant point tha.

Exploitation: Used parameter pollution to view other customers' orders and personal information.

Doharpan: Doosre grahakon ke aadesh aur vyaktigat jankari dekhne ke liye parameter pollution ka upyog kiya gaya.

Impact: Exposure of customer data including addresses, payment information, and order history.

Prabhav: Grahakon ke pata, bhugtan jankari, aur aadesh itihas jaise data ka prakasha.

Secure Coding Practices

Surakshit Coding Abhyas

1. Implement Proper Authorization

Always check user permissions before granting access to any resource or functionality.

Kisi bhi srotak ya karyakshamata tak pravesh dene se pahle hamesha upyogkartri anumatiyon ki jaanch karein.

// Example: Check user role before allowing access function canAccessAdminPanel(user) { return user.role === 'admin'; } // Always verify ownership for user-specific resources function canAccessUserResource(currentUser, resourceOwnerId) { return currentUser.id === resourceOwnerId || currentUser.role === 'admin'; }

2. Deny by Default

Use a deny-by-default approach where access is explicitly granted only when permissions match.

Ek deny-by-default drishtikone ka upyog karein jahan kewal anumatiyon milne par hi pravesh spast rup se diya jata hai.

// Example: Deny access unless explicitly allowed function checkAccess(user, resource) { // Check if user has explicit permission if (hasPermission(user, resource)) { return true; } return false; // Deny by default }

3. Validate All Inputs

Validate and sanitize all user inputs, especially IDs and parameters used for resource access.

Sabhi upyogkartri inputs ki jaanch aur shodhan karein, vishesh roop se srotak pravesh ke liye upyog kiye jane wale IDs aur parameters.

// Example: Validate user ID format function isValidUserId(userId) { return /^\d+$/.test(userId); // Only numeric IDs allowed } // Example: Prevent parameter pollution function getSingleParam(req, paramName) { const param = req.query[paramName]; return Array.isArray(param) ? param[0] : param; }

4. Secure Session Management

Implement secure session management to prevent session hijacking and fixation attacks.

Session hijacking aur fixation attacks ko rokne ke liye surakshit session prabandhan lagu karein.

// Example: Regenerate session ID after login app.post('/login', (req, res) => { // Authenticate user if (authenticated) { req.session.regenerate(() => { req.session.userId = user.id; res.redirect('/dashboard'); }); } });
